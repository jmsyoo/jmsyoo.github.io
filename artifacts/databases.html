Artifact Description

Database layer behind verification and selling flows. Examples are sanitized with fake data.

Improvements

  • Integrity & Safety: unique SerialNumber; CHECK constraints for enums/amounts; strong FKs; optional optimistic locking.
  • Performance: targeted indexes on hot paths; parameterized queries; bounded IN-lists via batching upstream.
  • Transactional Correctness: single transaction + row locks; optional idempotency keys for safe retries.

Outcomes & Reflection

This demonstrates software engineering/database outcomes and a security mindset. The hardest parts were concurrency and choosing practical indexes; I kept the set minimal yet aligned to access patterns.
